# 專案未來發展建議與規劃

這份文件整理了針對 `tradiagram` (台鐵運行圖) 專案的技術調整、功能新增與發展方向建議，供團隊討論使用。

## 1. 技術架構現代化 (Refactoring)
*目標：降低維護成本，提升開發效率與效能。*

- [ ] **前端框架升級**：
  - 現狀：jQuery + Bootstrap 3 (較舊)。
  - 建議：遷移至 **React**、**Vue** 或 **Svelte**。
  - 優點：更適合處理大量列車數據與複雜的狀態管理。
- [ ] **UI 介面升級**：
  - 建議：升級至 **Bootstrap 5** 或改用 **Tailwind CSS**。
  - 優點：改善行動裝置支援，介面更現代輕量。
- [ ] **繪圖效能優化**：
  - 現狀：SVG (DOM 元素多時易卡頓)。
  - 建議：若遇效能瓶頸，改用 **Canvas API** (如 PixiJS, Konva.js)。

## 2. 新增殺手級功能 (New Features)
*目標：增加工具實用性，從「靜態查詢」轉為「動態監控」。*

- [ ] **即時誤點顯示 (Real-time Overlay)**：
  - 功能：結合台鐵即時動態 API，在計畫運行線旁繪製「實際運行線」。
  - 價值：一眼看出誤點與塞車路段。
- [ ] **互動式查詢增強**：
  - **Hover 資訊**：滑鼠懸停顯示車次、車型、起訖站。
  - **車次搜尋**：輸入車次 (如 408) 直接高亮定位。
  - **車站時刻表**：點擊車站名顯示該站詳細時刻。
- [ ] **深色模式 (Dark Mode)**：
  - 功能：提供黑底螢光線條的配色。
  - 價值：符合專業監控系統習慣，降低長時間觀看的眼睛疲勞。

## 3. 自動化與維運 (DevOps)
*目標：減少人工介入，確保資料即時性。*

- [ ] **GitHub Actions 自動更新**：
  - 功能：設定每日排程，自動執行爬蟲腳本抓取新資料並 Commit。
  - 價值：網站資料永遠保持最新，全自動化運作。
- [ ] **資料傳輸優化**：
  - 功能：在後端/腳本階段對 JSON 進行瘦身，只保留繪圖必要欄位。
  - 價值：減少使用者流量消耗，加快載入速度。

## 4. 使用者體驗 (UX)
*目標：擴大使用場景，優化操作手感。*

- [ ] **行動裝置適配**：
  - 建議：引入 `svg-pan-zoom` 等套件。
  - 功能：支援手機雙指縮放、拖拉平移。
- [ ] **PWA (Progressive Web App)**：
  - 功能：支援「安裝」至手機桌面、離線瀏覽舊資料。
